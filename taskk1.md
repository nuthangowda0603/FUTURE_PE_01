## 🧠 Prompt Engineering Workflow

### Master Prompt Framework
```
You are an expert website copywriter for local businesses.

Business Type: [BUSINESS TYPE]
Business Name: [NAME]
Location: [CITY]
Target Audience: [WHO THEY SERVE]
Tone: [TONE DESCRIPTOR]
USP: [UNIQUE SELLING POINT]

Write the following sections:
1. HOMEPAGE — Hero headline, subheadline, and 3 value propositions
2. SERVICES PAGE — 3 service descriptions (50–70 words each)
3. CTA SECTION — A compelling call-to-action block with headline, body, and button text

Rules:
- No jargon. Write like a human, not a brochure.
- Every line must earn its place — cut filler.
- Focus on benefits, not features.
- Use "you" language to speak directly to the reader.
- End every section with momentum toward conversion.
```

### Tone Adaptation Prompt
```
Rewrite the following copy in a [TONE] voice.
Original tone was [ORIGINAL TONE].
Keep the same structure but change:
- Word choice and sentence rhythm
- Level of formality
- Emotional register
- CTA urgency
```

## 🎯 Prompt Engineering Principles Applied

**1. Role-setting** — Always opening with "You are an expert copywriter for [context]"

**2. Audience anchoring** — Specifying who reads the copy shapes every word choice

**3. Tone as a parameter** — Describing tone in metaphors ("like a trusted friend") produces more consistent results than adjectives alone

**4. Constraint prompting** — Rules like "no jargon" and "use 'you' language" prevent generic output

**5. Output scaffolding** — Requesting specific sections (Hero / Services / CTA) in a numbered format ensures completeness

**6. Iterative refinement** — Using a second "tone adaptation" prompt to refine and tighten initial outputs
